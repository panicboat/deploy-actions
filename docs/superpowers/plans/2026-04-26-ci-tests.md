# CI Tests Introduction Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** PR の作成・更新時に RSpec / `workflow-config.yaml` 検証を自動実行する GitHub Actions workflow を導入する。

**Architecture:** `.github/workflows/check.yaml` を 1 ファイル新設し、2 ジョブ（rspec / validate-config）を並列実行する。`pull_request` のみトリガーし、`action-scripts/**`, `label-dispatcher/**`, `label-resolver/**`, workflow 自身のパスフィルタで関係ない PR ではスキップする。

**Tech Stack:** GitHub Actions, `actions/checkout@v4`, `ruby/setup-ruby@v1`, Ruby 3.4.5, Bundler, RSpec。

**Spec:** `docs/superpowers/specs/2026-04-26-ci-tests-design.md`

### Task 2: `.github/workflows/check.yaml` の追加

**Files:**
- Create: `.github/workflows/check.yaml`

- [ ] **Step 1: workflow ファイルを作成**

ファイル `.github/workflows/check.yaml`:

```yaml
name: check

on:
  pull_request:
    paths:
      - 'action-scripts/**'
      - 'label-dispatcher/**'
      - 'label-resolver/**'
      - '.github/workflows/check.yaml'

concurrency:
  group: check-${{ github.ref }}
  cancel-in-progress: true

defaults:
  run:
    working-directory: action-scripts

jobs:
  rspec:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: ruby/setup-ruby@v1
        with:
          ruby-version-file: action-scripts/.ruby-version
          bundler-cache: true
          working-directory: action-scripts
      - run: bundle exec rspec

  validate-config:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: ruby/setup-ruby@v1
        with:
          ruby-version-file: action-scripts/.ruby-version
          bundler-cache: true
          working-directory: action-scripts
      - run: bundle exec ruby config-manager/bin/config-manager validate
```

- [ ] **Step 2: YAML 構文チェック**

Run:
```bash
ruby -ryaml -e 'YAML.load_file(".github/workflows/check.yaml")' && echo OK
```

Expected: `OK` が表示される（パースエラーなし）。

- [ ] **Step 3: 各コマンドがローカルで通ることを確認**

Run:
```bash
cd action-scripts && bundle exec rspec && bundle exec ruby config-manager/bin/config-manager validate
```

Expected:
- rspec: `213 examples, 0 failures`（または現在のスペック数で 0 failures）
- validate-config: `✅ Configuration is valid` を含む出力で exit 0

- [ ] **Step 4: コミット**

```bash
git add .github/workflows/check.yaml
git commit -s -m "ci: add check workflow for rspec and validate-config"
```

### Task 3: PR 作成と CI 実行確認

**Files:** なし（GitHub 上での確認のみ）

- [ ] **Step 1: ブランチを push**

Run:
```bash
git push -u origin HEAD
```

Expected: `ci/introduce-checks` が origin に作成され、tracking が設定される。

- [ ] **Step 2: Draft PR を作成**

Run:
```bash
gh pr create --draft --title "ci: introduce checks workflow (rspec, validate-config)" --body "$(cat <<'EOF'
## Summary
- PR トリガーの GitHub Actions workflow `.github/workflows/check.yaml` を新設
- 2 ジョブ並列: rspec / validate-config

## Spec
docs/superpowers/specs/2026-04-26-ci-tests-design.md

## Test plan
- [ ] 2 ジョブが並列起動する
- [ ] 全ジョブ green
- [ ] README のみ修正したコミットで paths filter が効くことを確認
- [ ] 連続 push で concurrency による cancel が動くことを確認
EOF
)"
```

Expected: PR URL が出力される。

- [ ] **Step 3: CI 実行を確認**

Run:
```bash
gh pr checks --watch
```

Expected: rspec / validate-config の 2 チェックすべてが ✓（pass）になる。

落ちた場合:
- rspec: ローカルで通っている spec が CI で落ちる場合は環境変数依存・時刻依存などを疑う
- validate-config: `workflow-config.yaml` のパスや config-manager の依存に問題 → ローカル再現を試みる

- [ ] **Step 4: paths filter の動作確認**

Run:
```bash
echo "" >> README.md
git add README.md && git commit -s -m "docs: trigger paths filter test"
git push
```

`gh pr checks --watch` で、新しい push に対して check workflow が**起動しない**ことを確認する（GitHub UI では "Expected — Waiting for status to be reported" にならず、そもそも Actions タブに新規 run が現れない）。

確認後、テスト用コミットを取り消す:
```bash
git reset --hard HEAD~1 && git push --force-with-lease
```

注: `--force-with-lease` は自分の最新ブランチ前提でのみ使う安全な force push。

- [ ] **Step 5: concurrency の動作確認**

Run:
```bash
git commit --allow-empty -s -m "ci: trigger run A"
git push
git commit --allow-empty -s -m "ci: trigger run B"
git push
```

GitHub Actions タブで、最初の run（"trigger run A" に対応）が "Cancelled" になり、2 つ目の run（"trigger run B"）のみが走り続けることを確認。

確認後、テスト用コミットを取り消す:
```bash
git reset --hard HEAD~2 && git push --force-with-lease
```

- [ ] **Step 6: PR を Ready for review に変更**

Run:
```bash
gh pr ready
```

Expected: PR が Draft から Ready for review に変わる。
